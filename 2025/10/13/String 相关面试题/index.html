String 相关面试题

2025-10-15 · 5 min · 1.3k words · Java 基础

1. String 是什么？能继承吗？

  • String 是 final 修饰 的类，不可继承。
  • 内部使用 char[]（JDK 9+ 用 byte[]） 存储字符。
  • 不可变（immutable） 的类。

2. 为什么 String 是不可变的？

  • 底层 value[] 被 final 修饰。
  • 没有提供修改内部数组的接口。
  • 安全、线程安全、缓存 hashcode、String Pool 优化等原因。

3. String、StringBuffer、StringBuilder 区别？

特性 String StringBuilder StringBuffer
可变性 不可变 可变 可变
线程安全 ✅ 安全（不可变） ❌ 不安全 ✅ 安全（加锁）
性能 慢 快 较慢
适用场景 少量字符串拼接 单线程拼接 多线程拼接

6. "a" + "b" 与 new String("ab") 有什么区别？

  • "a" + "b" 在编译期优化为 "ab"（常量折叠）。
  • new String("ab") 会创建新对象，堆内存在常量池之外。

🧮 五、String 常量池与内存分布

10. String 存放在哪里？

类型 存储位置
字符串常量 方法区（JDK 8 前）/ 元空间（JDK 8 后）
new 创建的字符串 堆内存
intern() 后 常量池

11. String Pool 的生命周期？

  • 在 JVM 启动时加载。
  • JVM 退出时销毁。
  • 常量池中字符串 不会被 GC 回收（除非类卸载）。

🧱 七、String 的内存优化与安全性

13. 为什么 String 设计成不可变的？

  • 安全性（网络地址、文件路径、ClassLoader 等场景）
  • 线程安全
  • 可作为 HashMap 的 key
  • 可缓存 hashcode
  • 常量池复用节省内存
